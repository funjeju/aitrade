# Skill: backtest-guard

## 목적
백테스트는 **틀리면 조용히 사용자를 망하게 하는** 기능이다(CLAUDE.md P2). 코드가 아래 편향을 구조적으로 차단하지 못하면 머지하지 않는다.

## 통과 필수 체크리스트

### 1. 표본 분리 (필수)
- [ ] **out-of-sample 분리**가 되어 있는가? in-sample 수치 단독 노출은 금지.
- [ ] **walk-forward**(또는 최소 train/test 분리)가 기본값인가?
- [ ] 결과에 `split.method`, in/out 각각의 지표가 **분리 기록**되는가?

### 2. Look-ahead bias
- [ ] t 시점 판단이 **t 이후 데이터**를 참조하지 않는가? (지표·시그널·정규화 포함)
- [ ] 리샘플/롤링 계산에서 미래 정보 유입이 없는가?
- [ ] look-ahead를 주입하면 **실패하는 회귀 테스트**가 있는가?

### 3. Survivorship bias
- [ ] 유니버스에 상장폐지·거래정지 종목이 포함되는가?
- [ ] 지수/섹터 구성의 **과거 시점 구성**을 쓰는가(현재 구성 역투영 금지)?

### 4. 데이터 스누핑 / 과최적화
- [ ] 같은 검증셋에 대한 **재최적화 횟수(reoptimizationCount)**를 기록·경고하는가?
- [ ] "AI 개선 반복"이 **무제한**이 아닌가? 임계 초과 시 사용자 경고.
- [ ] 파라미터 수 대비 표본이 과소하지 않은가(경고)?

### 5. 현실성
- [ ] 수수료·슬리피지·체결 가정이 명시·반영됐는가?
- [ ] 유동성 부족 종목의 비현실적 체결이 없는가?

### 6. 표시 (컴플라이언스 P3)
- [ ] 모든 성과 수치에 **기간·표본수·검증방식**이 병기되는가?
- [ ] 미실현 수익률을 실현처럼, 유리한 기간만, 단정적으로 표시하지 않는가?

## 실패 처리
- 하나라도 미충족이면 **머지 금지**. 무엇이 왜 막혔는지 명시하고 수정 요구.
