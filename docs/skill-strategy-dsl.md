# Skill: strategy-dsl

## 목적
"말로 설명한 전략"을 엔진이 실행 가능한 **Strategy DSL(JSON)**로 손실 없이, 그리고 **컴플라이언스 안전하게** 변환한다.

## 절대 규칙
1. **사용자 의도를 그대로 실행하지 않는다.** 조건이 부족하면 **먼저 되묻는다**.
2. AI는 "이 종목을 사라"고 하지 않는다. 산출물은 **사용자 전략 조건의 형식화**일 뿐이다(CLAUDE.md P1).
3. 정량화한 모든 파라미터는 **어떤 공식으로 계산되는지 사용자가 볼 수 있게** 함께 반환한다(P4).

## 절차
1. **의도 파싱**: 자연어에서 [유니버스/섹터, 대장주 기준, 기준 캔들 조건, 진입(눌림/거래량/이평), 분할, 청산/손절]을 추출.
2. **결측 조건 탐지 → 되묻기**: 아래를 채우지 못하면 번호 목록으로 질문한다.
   - 장대양봉 기준(시가 대비 고가 %?)
   - "최근"의 정의(몇 거래일?)
   - 거래량 기준(전일 대비 vs 평균 대비, 배수?)
   - 눌림 허용오차(몸통 절반 ±%? 시가 부근 ±%?)
   - 이평선 기간·기울기 임계
   - 분할 매수 구간·비중
   - 청산/손절 기준(트레일링 방식·손절 근거)
3. **정량화 매핑**: `docs/04-strategy-engine.md`의 "정량화 사전"을 사용. 새 표현은 사전에 없으면 추가 제안 후 확인.
4. **DSL 생성**: `04-strategy-engine.md` §4.4 스키마로 직렬화.
5. **스키마 검증**: 필수 필드·타입·범위 확인. 실패 시 다시 되묻기.
6. **투명성 반환**: 각 파라미터 옆에 계산식 요약을 함께 제시.

## 되묻기 예시 형식
> 전략을 확정하려면 아래를 정해주세요:
> 1. 장대양봉 기준 — 시가 대비 고가 몇 % 이상인가요? (예: 20%)
> 2. "최근"은 며칠인가요? (예: 20거래일)
> 3. 거래량은 전일 대비 몇 배인가요? (예: 5배)

## 금지
- 결측 조건을 임의 기본값으로 **조용히** 채우기(반드시 확인 또는 명시적 표시).
- DSL에 근거 없는 종목명·개별 추천 삽입.
