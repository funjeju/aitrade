# 07. 로드맵 (Roadmap)

원칙: **규제 리스크·원가 리스크가 낮은 것부터, 가장 어려운 기술 난제(자연어→규칙, 정직한 백테스트)를 먼저 증명한다.** 자동매매·마켓플레이스는 뒤로 미룬다.

## Phase 0 — 기반 (Foundation)
- [ ] 모노레포 + Next.js(App Router) + TypeScript + Vercel 배포 파이프라인.
- [ ] Firebase Auth/Firestore/Functions 연결, custom claims(role) 골격.
- [ ] 디자인 시스템 토큰(다크/라이트) + 기본 셸(사이드바/탑바/테마·언어 토글).
- [ ] i18n(ko/en) 골격 + 누락 키 CI 검사.
- **완료 기준**: 빈 대시보드가 다크/라이트·ko/en에서 동일하게 뜨고 로그인·역할 가드가 동작.

## Phase 1 — 전략 설계 + 정직한 백테스트 (MVP 핵심)
> 자동매매·실시간 스캔 없이, **가장 어려운 두 난제**를 먼저 증명. 규제 리스크 최소.
- [ ] `strategy-engine`: 지표 정량화 함수 + 기준 캔들 로직 + 단위 테스트.
- [ ] AI Strategy Chat: 자연어→DSL, **부족 조건 되묻기**(`strategy-dsl` 스킬).
- [ ] 전략 버전 관리(불변 버전, 변경 요약).
- [ ] 백테스트: **out-of-sample/walk-forward 기본**, look-ahead·survivorship 가드(`backtest-guard`).
- [ ] 결과 화면: 승률/평균수익/MDD/보유일 + **검증방식·기간·표본 병기**.
- **완료 기준**: 사용자가 말로 전략을 만들고, 과최적화 방지가 내장된 검증 결과를 본다.

## Phase 2 — 실시간 스캐너 + 근거 시각화
- [ ] 데이터 벤더 확정·연동(라이선스·원가 확인 — 블로커).
- [ ] 스캐너 워커: 종목별 기준 캔들 상태 유지, 주기 스캔.
- [ ] Live Scanner Results + Alerts.
- [ ] Chart Preview: 기준 캔들·매수/손절/트레일 구간 **근거 오버레이**(지시 아님, P1).
- **완료 기준**: 활성 전략 조건에 부합하는 종목이 실시간 표시되고 차트에서 근거 확인.

## Phase 3 — 집행 (반자동 → 자동)
- [ ] 증권사 API 연동, **사용자 정의 조건 충족 시 주문 집행**(P1, 시스템트레이딩 형태).
- [ ] 반자동(승인형) → 자동 순으로.
- [ ] 체결 신뢰성·지연·리스크 컨트롤(손절 자동화).
- **선행 조건(블로커)**: 규제 검토 완료(로펌/비조치의견서), 집행 경로 신뢰성 검증.

## Phase 4 — 확장 (후순위, 규제 재검토 후)
- [ ] Marketplace(전략 공유·구독) — **유사투자자문/투자자문 해당 여부 확인 전 착수 금지**.
- [ ] 추가 언어(중/일 등), 추가 시장.

## 상시 병행
- 원가 모델 추적(유저당 스캔/데이터 비용), 성과 표시 컴플라이언스(P3), 감사 로그.

## 결정 대기 (로드맵을 흔드는 미해결 항목)
- 타깃 A(숙련자 도구) vs B(초보 대상) — 현 로드맵은 **A 전제**.
- 데이터 벤더/라이선스 — Phase 2의 블로커.
- 자동매매 규제 요건 — Phase 3의 블로커.
