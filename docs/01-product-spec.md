# 01. 제품 스펙 (Product Spec)

## 1.1 제품 정의

ATS-OS는 기존 HTS/MTS 조건검색기를 대체·확장하는 **AI 기반 전략 운영체제**다. 숙련 트레이더가 경험으로 체득한 매매 원리 — 시장 흐름 판단 → 섹터 자금 유입 확인 → 대장주 선별 → 눌림목 대기 → 분할 진입 → 추세 추종 매도 — 를 **AI와의 대화로 정량화**하고, **검증·감시·집행**까지 잇는다.

기존 조건검색기의 한계: 이동평균선 돌파·거래량 증가·RSI·MACD 등 준비된 지표의 단순 조합에 머문다. 실제 투자자의 다단계 사고(시장→섹터→종목→타이밍)를 담지 못한다. ATS-OS는 이 사고 과정 자체를 시스템으로 만든다.

## 1.2 사고 흐름 = 제품 파이프라인

```
[시장 분석] → [섹터 선별] → [대장주 선별] → [기준 캔들 인식] → [눌림목 정량 분석] → [진입/청산 규칙] → [백테스트/버전관리] → [실시간 스캔] → [차트 시각화] → [반자동/자동 집행]
```

각 단계의 정량화 공식·규칙은 `04-strategy-engine.md`에 있다.

## 1.3 핵심 기능 모듈

| 모듈 | 설명 |
|---|---|
| **AI Strategy Chat** | 자연어로 전략 설명 → AI가 부족 조건 질문 → 전략 규칙 확정 |
| **Strategies** | 전략 목록·버전 관리·성과 이력 |
| **Backtest** | 과거 데이터 검증. 승률/평균수익률/MDD/평균보유기간. out-of-sample 필수 |
| **Scanner** | 실시간으로 전략 조건에 부합하는 종목 탐색 (BUY/WATCH 등 상태) |
| **Alerts** | 조건 충족 시 알림 |
| **Trades** | 체결/포지션 관리 |
| **Marketplace** | (후순위) 전략 공유·구독 — 규제 재검토 필요 |
| **Admin** | 운영자 모드 (`06-admin-i18n.md`) |
| **Settings** | 계정·테마·언어·API 연동 |

## 1.4 "감각의 정량화" 원칙

숙련자의 주관적 표현은 모두 계산 가능한 데이터로 매핑한다.

| 사람의 표현 | 정량화 방식 |
|---|---|
| "20일선이 예쁘게 우상향이다" | 이동평균선 기울기(최근 N봉 회귀 계수 / 각도) |
| "거래량이 자연스럽게 줄었다" | 최근 평균 거래량 대비 감소율 |
| "양봉이 힘 있게 마감했다" | 시가 대비 고가 상승률 · 종가 위치 · 몸통 크기 · 윗꼬리 비율 |
| "몸통 절반까지 눌렸다" | 기준 캔들 (시가+종가)/2 대비 현재가 위치 |

**단, 정량화는 손실적(lossy)이다.** 어디까지 정확히 계산되고 어디부터가 파라미터 의존 근사치인지 사용자에게 투명하게 노출한다 (CLAUDE.md P4).

## 1.5 규제·컴플라이언스 라인 ⚠️

> 이 절은 제품 설계 가이드이며 법률 자문이 아니다. 실제 서비스 개시 전 **금융 전문 로펌 검토 및 금융위 비조치의견서**가 필요하다.

### 안전지대로 설계하는 근거
- 금융위 유권해석상, **사용자가 직접 매매 종목·수량·가격·시기 조건을 지정하고 시스템은 조건 충족 시 주문만 자동 집행**하는 시스템트레이딩은 자본시장법상 투자자문업·투자일임업에 해당하지 않는 것으로 본 사례가 있다. → **"사용자가 전략의 주인, 시스템은 집행 도구"** 포지셔닝의 근거.
  - 출처: 금융위·금감원 유권해석(스마트리밸런싱 주문예약 서비스 관련), casenote.kr 정리본.

### 위험지대 (넘으면 등록 대상)
- **AI가 특정 사용자에게 종목/시점을 골라주는 개별 자문** → 투자자문업(금융위 등록).
- **유료 회원제 + 양방향 채널로 개별 조언** → 2024.8.14 시행 개정 자본시장법상 유사투자자문업이 아닌 **정식 투자자문업자**로 규율된다. (양방향·유료·개별 조언 조합 주의)
  - 출처: 금융위 보도자료 「유사투자자문업자의 불건전영업행위 규율 자본시장법 시행」(2024.8.14).
- **손실보전·이익보장 약정, 수익률 허위/미실현/유리한 기간만 표시, 확실하다고 오인시키는 단정적 표현** → 표시·광고 규제 위반.
  - 출처: 금융위 보도자료(광고 규제 4개 유형).

### 설계 규칙 (위 근거 → 제품에 반영)
1. AI는 **"당신의 전략 조건에 부합하는 종목"**을 보여줄 뿐, **"이 종목을 사라"**고 하지 않는다.
2. 대장주 "선정"은 **사용자가 정의/승인한 랭킹 기준의 계산 결과**로 제시하며, 시스템의 재량 추천으로 표현하지 않는다.
3. 성과 표시에는 항상 기간·표본·검증방식을 병기한다.
4. Marketplace(전략 공유·판매)는 규제 재검토가 끝나기 전까지 착수하지 않는다 → `07-roadmap.md`에서 후순위.

### 확인 필요 (미해결)
- [ ] 대화형(양방향) AI 챗이 "개별성 있는 자문"으로 해석될 경계 — 로펌 검토 필요.
- [ ] 자동매매 집행을 자체 제공 vs 증권사 API 위임 시 각각의 등록 요건.
- [ ] Marketplace의 유료 전략 판매가 유사투자자문/투자자문에 해당하는지.
