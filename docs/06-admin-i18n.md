# 06. 어드민 모드 & 다국어 (Admin & i18n)

## 6.1 어드민 모드

### 권한 모델
- 역할은 **Firebase custom claims**로 관리: `user | admin | superadmin`.
- 클라이언트는 role을 못 바꾼다. 부여/회수는 **Cloud Function(서버) 전용**.
- 라우팅 가드: Vercel Edge Middleware에서 세션 토큰의 claim 확인 → `/admin` 접근 제어.

### 어드민 기능 (시안의 Admin 메뉴 구체화)
| 영역 | 기능 |
|---|---|
| 사용자 관리 | 목록/검색, 플랜 변경, 정지/해제, role 부여(superadmin만) |
| 전략 운영 | 신고/문제 전략 확인, (Marketplace 도입 시) 심사 |
| 시스템 지표 | 스캐너 처리량, 백테스트 큐 상태, 에러율, 데이터 지연 |
| 데이터 소스 | 벤더 연결 상태·쿼터·비용 모니터 |
| 공지/피처플래그 | 점검 공지, 기능 토글, 티어별 스캔주기 제어 |
| 감사 로그 | `adminAudit` 열람(불변) |

### 감사(Audit) 원칙
- 모든 어드민 변경 액션 → `adminAudit/{id}`에 `{ actorUid, action, target, before, after, at }` 기록. **삭제·수정 불가**.
- 규제 대응·내부통제 근거(P3와 연결: 성과·광고 관련 변경도 추적).

### 어드민 UI
- 일반 앱과 **같은 디자인 시스템/토큰**(다크·라이트) 사용. 별도 스타일 금지.
- 위험 액션(정지, role 변경)은 확인 다이얼로그 + 사유 입력 필수.

## 6.2 다국어 (i18n)

### 스택 & 구조
- `next-intl` (App Router 호환). 로케일 세그먼트 라우팅: `/{locale}/...`.
- 기본 로케일 `ko`, 2차 `en`. 구조는 **N개 언어 확장 가능**하게 설계(하드코딩된 2개 가정 금지).
- 메시지 파일: `messages/{ko,en}.json` (필요 시 `locales/{lang}` Firestore로 원격화 — 어드민에서 편집).

### 규칙
- **사용자 노출 문자열은 100% i18n 키 경유.** 코드에 리터럴 한국어/영어 금지(CLAUDE.md §6).
- 네임스페이스로 분리: `common`, `dashboard`, `strategy`, `backtest`, `scanner`, `admin`, `errors`.
- 복수형·보간은 ICU 메시지 포맷 사용.

### 로케일 의존 도메인 규칙 (중요)
- **숫자/통화/날짜**: `Intl.NumberFormat`/`Intl.DateTimeFormat`으로 로케일별 포맷. 저장은 raw, 표시만 포맷.
- **가격 등락색**: 로케일이 아니라 **사용자 설정(`priceColorMode`)**으로 분리 관리 — 한국 사용자라도 미국식(상승=녹색)을 원할 수 있음. 기본은 KR(상승=적색). (`03-design-system.md` 참조)
- **시장/거래시간·용어**: 시장별로 다름 → 로케일과 별개의 `market` 축으로 관리.

### 번역 운영
- 신규 문자열은 ko 필수, en은 CI에서 **누락 키 검사**로 강제(빠지면 빌드 경고/실패).
- 어드민 "공지/피처플래그" 등 동적 텍스트도 언어별 필드 보유.
